# 三方对比：tbpu 排版 vs 无 tbpu vs 多模态 LLM 识别

> 时间：2026-07-23
> 对象：/root/uploads/img6~img10（5 张母婴产品实拍，1600×2133）
> 引擎：PaddleOCR 3.x 官方 PP-OCRv6_medium（CPU）
> 三方定义：
> - **A = 使用 tbpu**（复用 `ImageTableAdapter.extract` 默认路径 + `process_ocr_lines` GapTree/ParagraphParse）
> - **B = 不使用 tbpu**（直接 `ocr.predict` + 按检测框中心 y 朴素 top-to-bottom 排序，绕过 tbpu）
> - **C = 多模态 LLM**（复用上一轮内置视觉解析，直接"看"图读出语义）

## 一、统计

| 图 | A(tbpu) 字符 | B(无tbpu) 字符 | B 行数 | OCR 耗时 | 备注 |
|----|-------------|---------------|--------|---------|------|
| img6 绿森林鳕鱼肝油 | 708 | 722 | 61 | 94.7s | 双栏 |
| img7 小葵花绿标白瓶 | 308 | 314 | 27 | 58.2s | 近单栏 |
| img8 酵素益生菌粉 | 206 | 213 | 30 | 55.0s | 含营养成分表 |
| img9 小葵花儿童乳粉 | 409 | 417 | 30 | 60.9s | 近单栏 |
| img10 益瑞达灰盒 | 291 | 297 | 36 | 64.8s | 含营养成分表 |

字符数 A 略少于 B（tbpu 合并了被切断的长行、过滤了极低置信碎片，更紧凑）。**B 行数远多于 A 的根因是：B 把跨行的长文本/表格行列切成独立行**。

## 二、A vs B · 阅读顺序与排版质量（核心差异）

### img6（双栏）— tbpu 决定性胜利

**B（无 tbpu，朴素 y 排序）把双栏绞在一起、营养成分数据被打散：**
```
每100g含
标志性成分及含量
【注册号】国食健注G20130383【执行标准】0/YC0021S
【生产企业】广东亿超生物科技有限公司
80mg
维生素A
址】汕头市潮汕路金园工业城13-07、13-08片
1.3mg
维生素D
【经销商】葵花药业集团重庆小葵花健康产业发展有限公司
6g
二十二碳六烯酸(DHA)
5.2g
二十碳五烯酸(EPA)
```
→ "每100g含 80mg 1.3mg 6g 5.2g" 被左右栏同高度行切成碎片，穿插企业信息，**完全不可读**。

**A（tbpu）保持阅读流：**
```
标志性成分及含量
维生素A 维生素D
二十二碳六烯酸(DHA) 二十碳五烯酸(EPA)
【适宜人群】免疫力低下者
每100g含 80mg 1.3mg 6g 5.2g
【不适宜人群】少年儿童、孕妇、乳母
```
→ 左栏（成分）读完再右栏（企业），营养成分"项目行 + 数据行"配对成一行。

### img8 / img10（营养成分表）— tbpu 聚合表头与数据

**B（无 tbpu）把表格逐行拆碎（img8 例）：**

→ "项目/每份/能量/61千焦/蛋白质/0克…" 每个单元格一行，**表格结构完全丢失**。

**A（tbpu）把表头行与数据行配对：**
```

项目能量蛋白质脂肪

```
（img10 同理：`项目能量蛋白质脂肪 / 碳水化合物钠` 作表头行，`每100克 1567千焦 / 0克 1.0克 90.0克 80毫克` 作数据行，`NRV% 19% 0% 2% 30% 4%` 作 NRV 行）

### img7 / img9（近单栏）— tbpu 收益有限

两图基本单栏，A 与 B **顺序接近**，差异仅在：B 把长行切成两行（如"葵花药业集团重庆小葵花健康产业发展 / 有限公司"），A 合并为一行。tbpu 在单栏场景下主要做**行合并**，排版增益小但不破坏可读性。

## 三、C（多模态）vs A/B（OCR 全系）· 完整性差异

C 复用上一轮视觉解析，补全了 OCR（无论有无 tbpu）都漏的语义内容：

| 图 | OCR(A/B) 漏检 | C（多模态）补全 |
|----|--------------|----------------|
| img6 | 顶部品牌艺术字切碎（"世然 S 中国年国宝宝体质"） | 识别为"绿森林/中国年国宝宝体质"品牌区，不当正文 |
| img7 | **顶部"若服用抗生素请间隔2小时"绿色提示整行丢** | ✅ 完整读出 |
| img8 | **【配 料】整段丢**（聚葡萄糖…LG12…）、食用方法、贮存条件丢、条码首位漏 | ✅ 完整读出配料/食用方法/贮存 |
| img9 | **顶部红色提示第 2、3 条丢**（燕窝酸≤500mg、黑褐点说明）、贮存条件丢 | ✅ 完整读出 |
| img10 | 顶部 YRD® logo 切碎（"yro R"）、NRV 表头标签串行 | ✅ 识别为 YRD® 益瑞达，表头结构完整 |

**OCR 全系（A 与 B）的共性短板**：艺术字/logo、彩色提示框长说明、密集配料表——这些都是字符级检测易漏/易碎的区域。
**tbpu 只解决"顺序/排版"，不解决"漏检"**——A 和 B 在完整性上几乎一致（tbpu 不改识别内容，只重排）。

## 四、三方定位结论

| 维度 | B 无 tbpu | A 使用 tbpu | C 多模态 LLM |
|------|----------|------------|--------------|
| 字符级精确（条码/注册号/批号） | ✅ | ✅ | ⚠️ 易幻觉改写 |
| 双栏/表格排版 | ❌ 绞碎不可用 | ✅ 配对成行 | ✅ 理解结构 |
| 阅读顺序 | ❌ 左右栏交错 | ✅ 合理 | ✅ 合理 |
| 艺术字/logo | ❌ 切碎 | ❌ 切碎 | ✅ 识别为品牌区 |
| 彩色提示/配料/长说明 | ❌ 漏检 | ❌ 漏检 | ✅ 完整 |
| 不编造/可审计 | ✅ 诚实搬运 | ✅ 诚实搬运 | ❌ 生成有幻觉风险 |
| 端侧成本 | 低 | 低（仅后处理） | 高（需 VL 模型） |

**结论**：
1. **B（无 tbpu）基本不可用**——对双栏/表格产品图，朴素 y 排序会把内容绞成碎片，tbpu 是 OCR 进库的**必备后处理**，不是可选项。
2. **A（tbpu）是 OCR 基线的必要组成**——它把 OCR 的字符级输出变成"可读、可入库"的文本，但**不补漏检**。
3. **C（多模态）补的是"A/B 都漏的语义层"**（配料、提示、logo），而非"排版"——排版已由 tbpu 解决。
4. **三者不是替代关系，而是分层互补**：
   - OCR + tbpu = **字符级精确 + 排版可读** 的事实基线（条码/注册号/成分数据必须精确）
   - 多模态 = **语义补全** 层（补 OCR 漏的艺术字/配料/提示）
   - 融合规则：**数字/编号类以 OCR 为准（防幻觉），描述类以多模态为准（补漏检）**

## 五、对当前工具的印证与下一步

- 当前 `image_table.py` 默认走 A（tbpu），方向正确——本次测试证明去掉 tbpu 会直接废掉双栏/表格图的可读性。
- 上一轮已指出、本轮再次印证的缺口：**OCR（含 tbpu）对艺术字/配料/彩色提示漏检**，需引入多模态补全层（P0）。
- 落地形态：把 `llm` provider 升级为**多模态 provider**，`structure()` 同时收「OCR 文本 + 图片」，prompt 明确"数字以 OCR 为准、描述类补 OCR 漏检、不得改写 OCR 事实"。
