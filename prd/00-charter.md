# 项目总纲（Charter）— 母婴垂类 ToB Agent

> 单一事实源（Single Source of Truth）。代码意图与本文冲突时，先更新本文，再改代码。
> 维护方式见 `02-index.md` 末尾的「维护规则」。
> 修订说明：v2 基于用户明确答复修订——微信定为**个人微信**、LLM **每企业可配置**、
> 知识来源为 **PDF/说明书 + 图片表格 + 爬虫**（无结构化 API）；并已移除此前我擅自编造的非目标。

## 意图（Intent）

为**母婴行业企业（ToB）**打造一个**垂直领域 AI Agent**：按每家企业的独特产品结构**定制知识库**，
配套**知识转化工具链**（爬虫 / OCR / 多源统一接口）把散落知识汇入同一接口；以**端侧部署**形态
（1 家企业 = 1 个 Agent 实例）交付；企业内**多名员工通过微信（个人微信）共用同一个 Agent 工作**，
且**每位员工的会话相互独立、互不串扰**。

## 目标（Goals）

- [ ] G1 知识转化：提供**爬虫 + OCR（图片/表格/扫描件 PDF）** + 多源统一适配接口，把企业散落的
      非结构化知识（PDF 说明书 / 产品图 / 表格 / 官网网页）转接到同一接口。
- [ ] G2 企业定制知识库：依据每家母婴企业的产品结构，生成企业专属可检索知识库（结构化 + 向量）。
- [ ] G3 Agent 核心：基于 RAG 的垂类问答，支持企业定制 prompt 与**每企业可配置**的 LLM provider。
- [ ] G4 会话隔离：多员工共用同一 Agent 实例，按「企业 × 员工 × 会话」三级隔离，单会话独立。
- [ ] G5 微信接入：**个人微信**接入——多名员工从各自个人微信向同一 bot 账号发消息，系统按发送方
      `from_user_id` 识别员工并路由到其独立会话。
- [ ] G6 端侧部署：标准化封装，1 家企业 1 个实例，配置驱动企业定制，实例间数据与配置隔离；
      **个人微信形态下，端侧实例须同时承载微信协议客户端运行环境**。

## 范围边界（Scope Boundaries）

- **平台**：服务端以 Linux 容器为主，目标运行在客户内网 / 边缘设备（端侧）。
- **技术**：优先 Python 生态（AI/RAG 工具链成熟）；**个人微信接入采用腾讯官方 iLink Bot API**
  （Hermes 源码已确认：标准开放协议、HTTP 长轮询，非逆向协议，封号风险显著低于协议方案）。
- **性能**：单实例并发以「数十名员工」为量级，非高并发互联网流量；响应延迟以可用为先。
- **合规**：母婴行业涉及婴幼儿健康建议，需明确「非医疗诊断」免责与内容安全边界（见各模块注意事项）。
- **数据**：企业知识库数据不出本实例（端侧），不回传第三方；**LLM 是否出网由每企业配置决定**
  （端侧本地 / 云 API 可切换）。

## 已确认决策（来自用户明确答复）

- C1 **微信形态 = 个人微信（官方 iLink Bot API）**：多名员工各自个人微信向同一 bot 账号发消息；
      员工身份 = iLink 消息中的 `from_user_id`。接入层是**官方开放协议、HTTP 长轮询（getUpdates）模型**
      （Hermes 源码确认，非 wechaty 逆向协议），故 O3 封号风险基本消解。
- C2 **LLM = 每企业可配置**：部署时按企业策略选择端侧本地模型或云 API，系统支持两种 provider 切换。
- C3 **知识来源 = PDF/说明书 + 图片/表格 + 爬虫**：均为非结构化来源，**暂无结构化 API 接入需求**
      （如后续出现商品库/ERP API，再作为新适配器扩充，不进入首版范围）。
- C4 **首版策略 = 先不写码**：本轮先定意图与方案、产出修订版 charter/架构，进入实现前需你确认。
- C5 **网关与 agent 核心 = 仅借鉴 iLink 契约自建轻量网关（O3′ 已定 方案 B）**：**不耦合 Hermes 运行时**。
      参考 Hermes `WeixinAdapter`（`gateway/platforms/weixin.py`）的 iLink Bot API 实现方式，自建：
      - 微信网关（HTTP 长轮询 `getUpdates` + `sync_buf` 续传 + `context_token` 回带 + 限流熔断 + 按 `message_id` 去重）；
      - 会话隔离（借鉴 `build_session_key` 的 `agent:main:weixin:dm:<user_id>` 思路，自研每员工独立会话）；
      - 配对绑定（借鉴 `PairingStore` / `dm_policy`，自研按实例的企业级员工绑定）；
      - **自研 RAG agent 核心**（MOD-agent），完全解耦。
      Hermes 仅作为「iLink 协议参考实现」来源，不作为依赖。
