# 检索回归修复说明（H4 / R4 / R2 / R7）

> 本次修复源于「分块 + 结构化预过滤」改造后引入的检索回归。核心结论：**FTS5 单字分词是导致跨域幻觉与分块精度双双失守的共同根因**，配套还修了 mock 嵌入的单字重叠缺陷与营销块（卖点）对排序的污染。

## 一、遇到的问题

| 用例 | 现象 | 期望 |
|------|------|------|
| **H4**（mock 闭环） | 问「你们卖汽车轮胎吗」竟推荐了「小鱼仔DHA藻油」 | 无命中，应告知暂无相关信息 |
| **R4**（bge） | 问「汽车轮胎」召回 5 条产品 | 跨域查询应 0 命中 |
| **R2**（bge） | 问「全营养/营养不良/挑食/偏瘦 奶粉」返回佳贝艾特（山羊奶粉） | 应返回小佳膳（全营养配方粉） |
| **R7**（bge） | 问「金领冠配方里用的什么奶基」返回其**卖点**块（含「配方」），而非**配料表**（含生牛乳） | 应命中配料表块，内容含「生牛乳」 |
| **R7**（bge） | 问「羊奶粉 DHA 含量」返回**配料表**块（无字面 DHA），而非**营养成分**块 | 应命中营养成分块，内容含字面「DHA」 |

## 二、根因分析

1. **FTS5 单字分词是跨域幻觉的元凶（H4 + R4）**
   原 `_fts_tokenize` 把中文拆成**单字** token。跨域查询「你们**卖**汽车轮胎」中的「卖」字，与产品「**卖点**」块里的「卖」字命中 → FTS 关键词召回不收阈值门控约束 → 直接返回。向量路径（mock/bge）即便被阈值挡掉，FTS 仍把跨域结果带回。

2. **mock 嵌入用单字 bigram（H4 向量路径）**
   原 `mock_embed` 对中文取单字 + 相邻 bigram。任意单字重合（如「卖」）都能产生假低距匹配，**与维度无关**，无法靠调阈值根治。

3. **卖点块是营销 boilerplate，污染分块精度（R7）**
   各产品的「卖点」块内容高度雷同（「0-6个月新生儿；肠胃娇嫩…关注脑视力发育」），bge 把它们嵌入成近似向量，导致：
   - 参数级查询（DHA / 奶基）被雷同的卖点块挤占，命中不到真正的规格块；
   - 「配方」一词出现在卖点中，使「奶基」查询命中卖点而非配料表。

4. **差异化信号没进主块（R2）**
   小佳膳的「全营养 / 营养不良 / 挑食 / 偏瘦」只存在于卖点块（被降权/被删前），主块（基础信息）缺这些关键词，bge 仅凭「奶粉+宝宝」把山羊奶粉排到了全营养粉前面。

5. **bge 对相近品类 borderline 排序（R2 另一例）**
   问「…1段**牛奶粉**」时，bge 因「新国标 / 1段」重叠把山羊奶粉排在了牛奶粉前面。

## 三、修复方案

1. **统一「领域词表」分词，彻底去掉单字**（embeddings.py + store.py）
   - 新增 `MATERNITY_LEXICON`，与 `CATEGORY_TERMS` 合并为 `_domain_tokens()`：只产出「品类词 + 复合词 + 英文数字词」，**不做单字 / bigram 切分**。
   - FTS 索引与查询改用同一套 `_domain_tokens`；跨域文本（如「汽车轮胎」）无任何领域词 → FTS 返回不匹配哨兵 `__OOV__`，从根上杜绝跨域误召回。
   - `mock_embed` 改为领域词袋；OOV 文本返回**固定正交哨兵向量**（保留维度，与所有领域向量维度不交）→ L2≈√2 > 门控阈值 1.405，被干净丢弃。

2. **删除卖点块，差异化关键词进基础信息块**（models.py + markdown_product.py）
   - `to_chunks` 去掉「卖点」块（营销 boilerplate 不再参与排序）。
   - 新增 `keywords` 字段，仅把来源侧标签/关键词（如「全营养 / 营养不良」）写入**基础信息**块；营销话术（highlights/特色配方）不再污染主块。

3. **显式实体/品类命中加权**（store.py）
   - 新增 `NAME_BOOST=2.5`：查询直接点名品牌 / 产品名 / 品类（如「金领冠」「牛奶粉」）时，该实体优先返回，纠偏 bge 对相近品类的 borderline 排序。
   - 分块类型加权仅保留对「卖点」的降权（已无卖点块，作安全兜底），其余规格块以 bge 原始相似度为唯一信号，保证细粒度参数查询自然命中对应块。
